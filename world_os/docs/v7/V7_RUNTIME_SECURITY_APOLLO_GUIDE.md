# V7 Runtime + Security + Apollo Guide

## Why V7 exists
V6 was broad and strong as an architecture/data/UI package, but the gap review showed the next move had to be runtime execution, security enforcement and Apollo-style state reconciliation.

## What V7 adds
1. Connector runner reference implementation.
2. Parser harness with standard acquisition envelope.
3. Quality gate runner.
4. Hash-chain audit writer.
5. Action engine with policy/approval handling.
6. Source health checker.
7. OPA/Rego policy model.
8. SPIFFE-style workload identity contract.
9. Secrets and export-control contracts.
10. Apollo desired-state schema.
11. Node-agent reference loop.
12. Reconciliation, rollout, health-gate and rollback modules.
13. AIP context builder and tool executor.
14. Foundry bitemporal object runtime schema.
15. Gotham case engine and event fusion rules.
16. Deployment reference files.

## What V7 still is not
It is not a live production cluster. It is the executable nucleus and control-plane reference pack that lets engineering start replacing documentation-only artifacts with working services.
