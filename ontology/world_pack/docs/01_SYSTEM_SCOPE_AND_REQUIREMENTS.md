# System Scope and Requirements

## Scope
The system creates a world-level operational ontology acquisition mesh. It maps each domain subject to:
- source families
- endpoint candidates
- document/OCR candidates
- benchmark/evaluation candidates
- subniche coverage
- ontology objects
- vector memory namespaces
- graph edge classes
- user and system interactions
- quality, policy and audit gates

## Non-goals
- No private surveillance.
- No interception of communications.
- No credential bypass.
- No terms-prohibited scraping.
- No restricted/classified tracking.

## Production acceptance
A subject row is production eligible only when:
1. source authority is recorded;
2. licence/terms are reviewed;
3. parser is implemented and tested;
4. schema and freshness checks pass;
5. provenance is complete;
6. ontology mapping is approved;
7. policy controls are enforced;
8. audit records are emitted;
9. vector memory is retrievable under policy;
10. graph edges are typed, weighted and evidenced.
