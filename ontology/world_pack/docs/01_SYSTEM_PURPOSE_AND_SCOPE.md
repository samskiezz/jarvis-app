# System Purpose and Scope

## Foundry-style purpose
Ingest, validate, transform, govern and operationalise world data into ontology objects, links, actions and workflows.

## Gotham-style purpose
Fuse entity, event, geospatial, temporal, sensor and document evidence into operational pictures, investigations, cases and mission workflows.

## Apollo-style purpose
Deploy, update, govern and monitor the connectors, parsers, services, policies, AI models and mission applications across cloud, edge, hybrid, restricted and offline environments.

## AIP-style purpose
Use governed ontology context, vector memory nodes, graph relationships, tools and action proposals under policy, audit and human approval.
